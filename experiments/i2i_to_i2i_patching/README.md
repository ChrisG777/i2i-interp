# i2i → i2i text-token activation patching

Captures activations from a **source** i2i run and patches the **text-token** slice into a **target** i2i run, sweeping across all 32 blocks. The two runs must share `instruction` (relax with `--allow-mismatched-instruction`) and must use a different ref (one of `real_ref_name` / `source_image_path` / `ref_seed`). `noise_seed` is not required to match — paper-scale pair sets deliberately use cross-seed pairings. Invariants are enforced by `I2IToI2IRunner.run_pair(source, target)`.

## Token slice patched

Slice boundaries come from the per-task `TokenLayout` (see [Patching framework](../README.md#patching-framework)).

| Category | MM block slice  | Single block slice           |
|----------|-----------------|------------------------------|
| `text`   | `txt_stream`    | `hidden[:, :text_end, :]`    |

`--text-token-mode` (repeatable; default `all`) selects which positions inside the text slice get patched:

| Mode           | Positions                                       | Flat-layout filename          |
|----------------|-------------------------------------------------|-------------------------------|
| `all`          | full 512-token slice                            | `patched.png`                 |
| `padding_only` | Qwen3 padding positions (complement of content) | `patched_text_padding.png`    |
| `content_only` | content positions (the real prompt tokens)      | `patched_text_content.png`    |

Pass any combination (e.g. `--text-token-mode all padding_only content_only`) to run multiple variants in the same per-pair dir, sharing reference + baselines. The per-mode skip means a re-run that adds a new variant only generates the new flat-filename PNG.

## Pair sets

Three families ship with checked-in `pairs.txt`, regenerable via [`build_pairs_color.py`](build_pairs_color.py), [`build_pairs_style.py`](build_pairs_style.py), and [`build_pairs_dreambench_humans.py`](build_pairs_dreambench_humans.py):

| Family | Pairs | Pairing rule |
|---|---:|---|
| `single9_4step_color` | 448 | 8 objects × 56 directed `(color_a, color_b)` per object; cross-seed `s_i` → `s_{(i+1)%5}`. |
| `mm7_4step_style_to_real` | 450 | 18 subjects × 5 prompts × 5 seeds; cross-seed style `s_i` → real `s_{(i+1)%5}`. |
| `mm7_4step_dreambench_humans` | 450 | 5 shared prompts × 90 directed `(subject_a, subject_b)` per prompt; cross-subject pairs use distinct `noise_seed`s. |

All three pass the runner's invariants without `--allow-mismatched-instruction`.

## Output layout

```
results_v4/i2i_to_i2i_patching/<source_task_id>__<target_task_id>/<run_timestamp>/
    task_metadata.json
    cli_args.json
    ref_source.png + ref_target.png
    source.png                # source i2i (with activation hooks)
    target_baseline.png       # target i2i with no patching
    target_t2i_clean.png      # target prompt + seed, no ref
    text_tokens/
        patched_mm0_to_mm0.png ... patched_single23_to_single23.png
        grid.png
```

Grid panel: `[Source, Target (baseline), Target t2i clean, <one cell per source block>]`. `task_metadata.json` records both `source_instruction` / `target_instruction` and `source_noise_seed` / `target_noise_seed` so asymmetric runs are self-documenting.

## Usage

```bash
# Single pair
uv run python experiments/i2i_to_i2i_patching/i2i_to_i2i_patch.py \
    --pair manual_solid_yellow_couch manual_solid_blue_couch

# Paper-scale color pair (cross-seed; same instruction, different ref)
uv run python experiments/i2i_to_i2i_patching/i2i_to_i2i_patch.py \
    --pair solid_yellow_couch_s0 solid_blue_couch_s1

# Stress: different subjects (requires --allow-mismatched-instruction)
uv run python experiments/i2i_to_i2i_patching/i2i_to_i2i_patch.py \
    --pair customize_property_style_dog_frisbee_s0 \
           customize_property_style_capybara_hotspring_s0 \
    --allow-mismatched-instruction
```

Pair lists live under [`pairs/`](pairs/) and are regenerated by the `build_pairs_*.py` scripts in this directory. See [experiments/README.md](../README.md#run--grade--tabulate) for the run → grade → tabulate workflow.
